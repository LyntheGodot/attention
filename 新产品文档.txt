一、目标人群：专注力低下人群以及任何依托电脑的进行学习、工作的人群。

二、技术栈与架构
当前实现：Python 3.x + tkinter（GUI）、pygame（音频）、pywin32 + psutil（Windows 窗口监测）
数据存储：本地 JSON 文件（config.json / stats.json / activity_records.json / category_mapping.json）
打包方案：PyInstaller（--windowed 无控制台模式）
项目结构：
  main.py                  # 入口
  gui/
    main_window.py         # 主菜单（三个入口按钮）
    pomodoro_window.py     # 核心番茄钟界面（进度圆环、控制按钮、白噪音、窗口监测调度）
    settings_window.py     # 双选项卡设置（计时选项 + 声音）
    stats_window.py        # 14天柱状图统计
    self_assessment_window.py  # 专注完成后的自我评估弹窗
  utils/
    timer.py               # 可变间隔随机提示计时器
    window_monitor.py      # Windows 前台窗口监测（1秒轮询）
    activity_storage.py    # 活动记录 JSON 存储（按日期+session_id 组织）
    category_mapper.py     # 应用→分类映射（自动推荐+用户训练）
    audio.py               # pygame 双通道音频（提示音 channel + 白噪音 music）
    config.py              # 配置管理（默认值+合并+验证）
    stats.py               # 每日专注时长统计

核心数据流：
  计时开始 → WindowMonitor.start() 1秒轮询前台窗口
  → 记录窗口标题/进程名/停留时长/切换次数
  → 计时结束 → WindowMonitor.stop() 返回活动列表
  → 弹出 SelfAssessmentWindow → 用户标注分类/分心/备注
  → 保存至 ActivityStorageManager（activity_records.json）
  → 同步更新 CategoryMapper（下次同应用自动推荐分类）

当前白噪音支持类型：森林（Forest white noise.mp3）、雷雨（thunderstorm2.mp3）
提示音文件：sound1.wav（自定义），备选方案为 Windows Beep 蜂鸣


二、核心功能：
2.1随机番茄钟：结合行为心理学可变间隔强化机制与神经科学理论，解决传统固定时长番茄钟对目标人群存在的预判焦虑、枯燥易放弃、适配性差的核心痛点。基于以下心理学原理：
1. 可变比率强化：随机时间的提示音和奖励机制，比固定间隔更能强化行为；用户会期待自己“roll”出的提示音时间点，从而将大的、枯燥的专注时段拆解为未知的、可变的不确定时段，帮助用户进入心流（当然有人会问提示音会不会影响心流，但实际体验下来，进入心流后的大部分情况是：心流带来的专注的紧迫感反而可能让用户忽略提示音，相当于跳过了这一次随机事件进入到下一段的专注中。另外，也可以在参数设置中将提示音次数随机化，或者开启纯净模式
2. 正念冥想：微休息时间引导注意力回收，提升元认知能力
功能详述：
主要：传统番茄钟的工作—休息时间占比是固定的，容易让人倦怠；随机番茄钟通过内在生成随机的提示音时间点，通过该时间点重新划分工作-信息比例。
其他：图形可视化进度的番茄钟：进度圆环实时显示专注进度；支持白噪音陪伴（ 双声道独立控制：提示音和白噪音互不干扰）；支持自定义参数设置：自定义专注/休息/间隔/微休息时长、提示音音量、白噪音类型、提示音次数
2.2分心状态识别：
功能详述：监测获取活跃窗口信息（不是后台进程，应该是当前打开的最上层展示的页面）。然后先别做分心拐点识别引擎，而是用我这个自我评估引擎：在用户任务进行时同步记录窗口信息（包括窗口切换跳转、停留时间等）在任务结束后（休息时间结束后）展示当次你记录的信息，并让用户自己勾选，哪些属于”分心行为”，用户标注后，将记录的窗口信息与用户标注信息一并记录到长期数据库中（本地JSON存储专注数据），方便后续：一、大模型ai分析数据；二、下一次如果再浏览跳转”分心行为”窗口 会跳出提醒

当前实现状态：
✅ 窗口监测模块（window_monitor.py）：基于 win32gui + psutil，1秒轮询前台窗口，记录进程名、窗口标题、域名提取、停留时长、切换次数
✅ 自我评估弹窗（self_assessment_window.py）：专注结束后自动弹出，展示本次所有窗口活动（按时长降序），支持：
  - 自动分类推荐（基于历史标注的 category_mapping.json）
  - 用户手动选择分类（工作/学习/社交/娱乐/浏览/其他）
  - 勾选”分心”标记
  - 添加备注
  - 实时计算分心占比百分比
  - 过滤 ≤6秒 的无效窗口（临时弹窗、右键菜单等）
✅ 活动存储（activity_storage.py）：JSON 本地存储，数据结构如下：
  {
    “YYYY-MM-DD”: [{
      “session_id”: “uuid”,
      “focus_duration”: 60,
      “activities”: [{
        “app_name”: “msedge.exe”,
        “window_title”: “原始标题”,
        “domain”: “提取的域名”,
        “duration”: 82.0,        // 分钟
        “switch_count”: 42,
        “window_category”: “📝 工作”,
        “is_distracting”: false,
        “user_notes”: “”
      }],
      “distraction_ratio”: 0.0,
      “total_switches”: 318,
      “average_focus_streak”: 16.1
    }]
  }
✅ 分类映射（category_mapper.py）：自动学习——用户每次标注后，应用名/域名→分类的映射自动保存，下次同应用自动推荐分类

===== AI复盘具体实现路径 =====

阶段一：本地数据预处理层（无需网络，离线可用）
  输入：activity_records.json（按日期聚合所有 session）
  输出：结构化统计报告（JSON → 自然语言模板填充）
  指标：
    a) 专注时长趋势：日/周/月专注总时长折线
    b) 分心画像：哪些应用/域名被标记为分心最多？分心行为集中在什么时间段？
    c) 高效时段识别：一周中哪天/一天中哪个时段专注质量最高（分心占比最低）
    d) 切换频率：平均每分钟切换几次窗口？高切换频率是否与分心占比正相关？
    e) 专注耐力：单次连续不切换窗口的最长时长（average_focus_streak 的变化趋势）
  实现方式：纯 Python 统计计算，不依赖 LLM，数据脱敏后可直接在前端展示

阶段二：LLM 复盘生成层（调用云端 API 或本地模型）
  技术方案选择：
    方案A（推荐初期）：云端 API 调用（OpenAI / Claude / 国内通义千问/DeepSeek）
      - 优势：零部署成本，模型能力强
      - 注意：用户数据需脱敏——只传应用分类名而非原始窗口标题
    方案B（隐私优先）：本地小模型（Ollama + Qwen2.5-7B / Llama-3.1-8B）
      - 优势：数据不出本机，隐私安全
      - 劣势：需要用户本地部署（约 4-8GB 显存/内存）
    方案C（混合）：本地做数据聚合 + 脱敏，云端仅收统计摘要做自然语言生成

  提示词工程设计（核心）：
    System Prompt 要素：
      - 角色：你是专注力教练，基于认知科学与行为心理学
      - 输入格式：JSON 结构化的专注数据（脱敏后）
      - 输出要求：
        1. 总体评价（1-2句，温暖鼓励的语气）
        2. 关键发现（列出2-3个数据洞察，如"你周三下午的专注质量最高"）
        3. 风险预警（如"最近3天分心占比持续上升"）
        4. 行动建议（具体的、可执行的一条建议，如"建议把重要任务安排在上午9-11点"）
        5. 认知科学小贴士（随机一条，如"番茄钟工作法的科学原理..."）
      - 语气要求：非评判性、鼓励式、具体而非空泛

  数据脱敏策略（重要）：
    ❌ 不传给 LLM：原始窗口标题（可能含个人信息）、完整域名、应用进程名
    ✅ 只传：分类统计（如"社交类占35%"）、时间段分布、数值型指标

  实现步骤：
    1. 在 utils/ 下新增 ai_analyzer.py：
       - DataAggregator 类：按日/周/月聚合 activity_records.json
       - PromptBuilder 类：将聚合数据 + 用户偏好 → 构造 prompt
       - LLMClient 类：封装 API 调用（支持多种 provider）
    2. 在 gui/ 下新增 ai_review_window.py：
       - 周期选择（今日/本周/本月/自定义）
       - 展示预处理统计图表 + AI 生成的自然语言复盘
       - 支持一键导出复盘报告（Markdown/PDF）
    3. 配置项扩展：
       - ai_provider: "openai" | "claude" | "deepseek" | "local"
       - ai_api_key: 加密存储（使用 keyring 库或系统凭据管理器）
       - ai_model: 具体模型名
       - privacy_mode: "strict"（仅统计摘要）| "full"（含分类标签名）

阶段三（进阶）：小样本学习 + 用户偏好适配
  - 用户对 AI 建议的反馈（有帮助/没帮助）作为微调信号
  - 逐步学习用户的沟通风格偏好（喜欢数据型还是鼓励型）
  - Few-shot 示例库：预置 5-10 条高质量复盘示例作为 prompt 中的 few-shot

阶段四（远期）：RAG 知识库增强
  - 构建专注方法论知识库（认知科学论文摘要、番茄钟变体研究、ADHD 行为干预指南）
  - 使用向量数据库（ChromaDB 本地/SQLite-vec）存储知识片段
  - 检索增强：根据用户近期专注模式，检索最相关的建议知识片段注入 prompt


2.5多样化番茄钟形式：
功能详述：暂设有记忆模式、激励模式、休养模式、纯净模式；不同模式番茄钟随机时间点触发后的对应功能不同：

记忆模式：考虑结合Anki卡片形式，及时录入用户的记忆需求或突如其来的灵感；（能否接入obsidian？）
  - 随机提示触发后弹出迷你输入窗口，快速记录灵感/待办
  - 支持接入 Obsidian（通过 Obsidian URI 协议 obsidian://new?vault=xxx&content=xxx 直接写入每日笔记）
  - 支持导出为 Anki CSV 格式

激励模式：旨在探索多样化反馈（Award）形式，激励用户，形成正反馈；但有效的激励反馈形式仍有待探索，目前想到的一个方案是：让大模型对记忆模式中的灵感关键词进行发散式内容生成，再回馈给用户，力求达到”突破用户认知边界”的效果

===== 多样化 Award 形式探索 =====

A. 认知激励类（零成本、高价值）：
  1. AI 灵感发散：用户标记的关键词 → LLM 生成跨学科联想
     例如：用户标记”傅里叶变换” → AI 回复”你知道吗？傅里叶变换的原理也被用于音乐推荐算法中，Spotify 用频谱分析来理解歌曲的'情绪'”
     效果：打破知识茧房，建立意外连接，触发好奇心
  2. 知识卡片彩蛋：随机掉落一条冷知识/名言（来自预置库+AI生成）
  3. “你的进步”数据卡片：AI 生成一句话总结近期进步（如”本月你的平均专注时长比上月提升了12%”）

B. 感官激励类：
  4. 音效收藏集：完成不同成就解锁新的提示音/白噪音（如雨林、篝火、咖啡馆、海浪等）
  5. 视觉主题解锁：专注时长累积解锁新配色主题/进度圆环动画风格
  6. 微休息引导语多样化：随机播放不同的正念引导语（预录音频或 TTS 生成）

C. 游戏化激励类：
  7. 专注货币系统：专注时长 → 金币，可用于”购买”主题/音效/桌宠装饰（零真实货币）
  8. 肉鸽式随机事件：专注过程中随机触发正面事件（如”你已连续专注30分钟！获得双倍金币卡”）
  9. 专注连胜（Streak）系统：连续 N 天达到目标 → 特殊勋章 + 奖励加成
  10. 每日挑战：随机任务如”今天尝试比昨天多专注5分钟”或”今天只打开3个以内应用完成工作”

D. 社交激励类（远期）：
  11. 好友专注 PK：比较专注时长但不暴露具体内容
  12. 匿名鼓励信：随机收到另一位用户的鼓励话语
  13. 专注公会/自习室：小型群组共同计时，互相监督

E. 实物/外部激励桥接（可选）：
  14. 接入 Habitica / Forest 等习惯养成应用的 API
  15. 生成”专注周报”精美卡片图（可分享到社交媒体）

关键设计原则：
  - 可变比率强化：奖励不是每次都有，而是随机掉落（概率 30-50%），强化”期待”本身
  - 避免过度游戏化：奖励是辅助而非目的，不能喧宾夺主
  - 个性化：根据用户历史数据调整奖励类型偏好（喜欢数据型还是趣味型）

休养模式：随机时间点触发后提示用户进行短暂的放松，引导短暂正念休息，降低认知负担
  - 随机触发眼部放松引导（20-20-20法则：每20分钟看20英尺外20秒）
  - 可选引导语：身体扫描/呼吸练习/颈部放松
  - 背景色逐渐过渡为暖色调，降低屏幕蓝光刺激

纯净模式：无随机触发时间点，类似普通番茄钟
  - 保留进度圆环和白噪音
  - 适合已经进入深度心流的场景（用户主动切换到纯净模式避免被打扰）

===== 模式切换机制建议 =====
  - 番茄钟开始前让用户选择模式（默认记忆上次选择）
  - 运行中支持一键切换模式（如进入心流后切换到纯净模式）
  - 不同模式的进度圆环颜色不同，视觉上区分

四、当前配置系统与数据模型

config.json 配置项：
  focus_time: 40          # 专注时间（分钟），默认40
  break_time: 5           # 休息时间（分钟）
  min_interval: 5         # 随机触发最小间隔（分钟）
  max_interval: 40        # 随机触发最大间隔（分钟），不能超过专注时间
  micro_break: 60         # 微休息时长（秒），默认60
  break_countdown: true   # 休息倒计时开关
  alert_count: 1          # 提示音播放次数（1-5次）
  volume: 0.5             # 提示音音量（0.0 - 1.0）
  white_noise_volume: 0.5 # 白噪音音量（0.0 - 1.0）
  white_noise_type: "forest"  # 白噪音类型: "forest" | "thunderstorm"
  window_monitoring_enabled: true  # 窗口监测开关

双声道设计：提示音使用 pygame.mixer.Sound（channel 播放），白噪音使用 pygame.mixer.music（独立 music 流），两者互不干扰。
